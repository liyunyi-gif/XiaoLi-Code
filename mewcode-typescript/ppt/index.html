不只是聊天记录
是结构化的状态机

ConversationManager 用一个 Message[] 数组承载完整的对话状态。每条消息不是纯文本——它保留了工具调用的 block 结构,让 LLM 能精确理解历史上下文。

消息类型

assistant
文本 + ToolUse + Thinking

输出 + isError + elapsed

MEWCODE.md + 记忆注入(首轮)
关键 API
addAssistantFull() → text + thinking + toolUses 一次写入
addSystemReminder() → hook通知/MCP指令/记忆召回
injectLongTermMemory() → 首轮注入 CLAUDE.md + MEMORY.md
replaceWithCompacted() → 压缩后"摘要+原文尾"重建

CONTEXT MGMT · 上下文管理
08 / 9

对话无限长
的秘密

LLM Context Window 有限(64K~200K token),但对话可能是无限的。XiaoLiCode 用三层机制解决这个问题。

AUTO-COMPACT

自动压缩触发

manageContext() 每轮开始时检查:当前 token 数 ≥ ContextWindow - MaxOutput 时触发压缩。压缩后 usage anchor 重置,避免对压缩后的消息列表使用旧基线。

ContextTooLongError → forceCompact → 重试当前 turn
SUMMARY

分级压缩策略

① LLM 生成对话摘要(保留关键决策+上下文)
② 保留 N 条最近原文(工具调用/结果不拆)
③ replaceWithCompacted() 用"摘要 + 原文尾"重建会话

RecoveryState 快照 Recent ReadFile → 压缩后回放
PERSIST

会话持久化

每条消息实时写入 .mewcode/sessions/{id}.jsonl。压缩边界标记 compact_boundary。/resume 恢复时回溯压缩态(摘要+原文尾),而非全量历史。

ToolResult budget → 大输出 spill 到磁盘,防内存爆炸
